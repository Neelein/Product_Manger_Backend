## Design decisions
- Image upload via multipart/form-data, saved to `/Users/neal/Desktop/Project/Product_Manger/media/images/announcements/` with UUID filename
- Image served via `/media/` FileServer path prefix
- List returns paginated results (default page=1, limit=20), ordered by created_at DESC
- Hard delete (no soft delete)
- publisher_name fetched via JOIN with members table in SQL functions

## 2026-07-19: Refactor rows.Next() loops to pgx.CollectRows

### Modified files
- `src/database/product_repo.go` — `List` and `GetPricesByDetailID` now use `pgx.CollectRows` instead of manual `for rows.Next()` + `append`
- `src/database/inventory_repo.go` — `ListInventories` and `ListItemsByInventoryID` now use `pgx.CollectRows`; `scanInventory` parameter changed from `pgx.Row` to `pgx.CollectableRow` for compatibility
- `src/database/announcement_repo.go` — `List` now uses `pgx.CollectRows`

### Design decisions
- `pgx.CollectRows` chosen over `pgx.ForEachRow` because it handles the loop, append, close, and err check in one call
