# List Members API → Room Member Availability

## Summary
Changed `POST /api/members` from listing all members to listing members who are NOT yet in a given chat room, for use in the "add members to room" UI.

## Changes
- Migration `011_list_members.up.sql`: replaced `list_members`/`count_members` with `list_members_not_in_room`/`count_members_not_in_room` (takes `room_id` parameter, excludes room members)
- `src/domain/chat.go`: added `RoomMembersRequest { room_id, page, limit }`
- `src/domain/member.go`: removed `ListMembersRequest`
- `MemberRepository` interface: `ListMembers` → `ListMembersNotInRoom`, `CountMembers` → `CountMembersNotInRoom`
- `MemberRepositoryPGX`: updated both implementations
- `ListMembers` handler: parses `RoomMembersRequest`, validates `room_id`, calls new repo methods
- Tests: room-aware listing, missing `room_id`, unauthorized
- `cleanupMembers` test helper: now truncates chat tables before deleting members to avoid FK constraint violations
