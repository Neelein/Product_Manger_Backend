# 2026-07-21: GitHub Actions CI and test fixes

## Changes

### CI pipeline
- Created `.github/workflows/ci.yml` with two jobs:
  - `unit`: runs `go test ./src/...` on push to `dev` and PR to `main`
  - `integration`: runs `go test -tags=integration ./src/test/...` with a PostgreSQL 16 service container (using `POSTGRES_DB=productdb_test`, health check with `pg_isready`)

### Bug fixes discovered during CI setup
1. **Migration `012_create_chat.sql`** — `list_chat_rooms_by_member` referenced `rc.id` but `read_receipts` has no `id` column (composite PK). Changed to `COUNT(*) FILTER (WHERE rc.message_id IS NULL)`.
2. **Database test `dropTables`/`runMigration`** — did not include chat tables or `012_create_chat.sql`. Added them so the database test package can run standalone.
3. **Chat repo test `createTestMemberForChat`** — hardcoded email `test@example.com` caused duplicate key errors when a test needed 2+ members. Changed to use a counter-based unique email.

## Motivation
Without CI, there was no automated check that tests pass before merging. The test bugs were latent because tests only ran manually in local Docker.
