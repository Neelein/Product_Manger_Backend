# CI/CD Deployment with Docker and GitHub Actions

## Summary
Set up automated deployment pipeline for the Product Manager Go backend using Docker, GitHub Actions, and golang-migrate.

## Changes

### Infrastructure
- Added `Dockerfile` with multi-stage Go build (golang:1.25-alpine → alpine:3.19)
- Added `.github/workflows/deploy.yml` — auto-deploy on push to `main`
- Server port changed from 8080 to 8090

### CI/CD Pipeline
- GitHub Actions builds Docker image and pushes to `ghcr.io/neelein/product_manager_backend`
- SSH into production server, pull image, restart container
- Connection to existing PostgreSQL container (`my_postgres_db`) via Docker network `my_app_net`
- First deploy auto-creates `productdb` database

### Database Migrations
- Migrated from manual SQL file execution to `golang-migrate` v4 with `pgx/v5` driver
- Migration files renamed from `*.sql` to `*.up.sql` for golang-migrate compatibility
- Embed migrations into Go binary via `//go:embed`
- Migrations run automatically on container startup before HTTP server starts
- Uses `schema_migrations` table to track applied versions

### Migration Cleanup
- Removed `004_alter_product_details` and `005_remove_products_description` (development artifacts incompatible with fresh schema)
- Renumbered migrations 006~012 → 004~010 to close gaps
- `schema_migrations` table on production needs to be reset: `DROP TABLE IF EXISTS schema_migrations;`
- Test files updated to use renumbered paths

# Chat Room Refactor: Remove Type Field

## Summary
Removed the `type` (direct/group) field from chat rooms to simplify the model. Users create a room with just a name, then add members via the existing AddMembers endpoint.

## Changes

### Migration SQL (`db/migrations/010_create_chat.up.sql`)
- Removed `type` column from `chat_rooms` table creation
- Removed `p_type` parameter from `create_chat_room` SQL function
- Cleaned up `ALTER TABLE DROP COLUMN type` (no longer needed since column is never created)

### Domain (`src/domain/chat.go`)
- Removed `Type string` field from `ChatRoom` struct
- Removed `Type` from `RoomResponse` JSON output
- Simplified `CreateRoomRequest` to only contain `Name` (removed `MemberIDs`)
- `IsMember` is now always true for the creator in responses

### Repository interface (`src/domain/repository.go`)
- Updated `CreateRoom` signature: removed `memberIDs []string` parameter
- Callers now add members explicitly via `AddMembers`

### Repository implementation (`src/database/chat_repo.go`)
- `CreateRoom` calls `create_chat_room(p_name, p_created_by)` without type
- Creator is automatically added as a member after room creation
- Updated `list_chat_rooms_by_member` SQL call

### Handler (`src/api/chat_handler.go`)
- `CreateRoom` handler simplified: no `member_ids` parsing, no type validation
- `RoomResponse` no longer includes `Type` field

### Tests
- Updated `CreateRoomRequest` usage in tests to only pass `Name`
- Removed `Type` assertions from test expectations
- Updated `CreateRoom` calls to match new interface (no memberIDs)

### Scope creep caught and reverted
- Sub-agent modified `middleware.go` to disable session rotation for development — **reverted** as out of scope
